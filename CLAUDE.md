This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

A collection of Python finance screening utilities focused on Canadian (TSX) equities. All scripts fetch live data from Yahoo Finance via `yfinance`. No tests, no build step.

## Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt          # yfinance, pandas, numpy
pip install tabulate                     # optional: prettier output in wheel_screener
```

## Tests

```bash
python -m pytest py/test_value_universe.py -v

# Both suites together
python -m pytest py/test_swing_universe.py py/test_value_universe.py -v
```

## Running the scripts

All scripts with hard-coded relative paths (`../data/`, `../out/`) expect to be run from the `py/` directory. `tickers_info.py` takes CLI args so it can be run from root.

```bash
# Swing universe builder (outputs to ../data/can_tickers_swing_universe by default)
cd py && python swing_universe.py

# Swing universe with custom paths
cd py && python swing_universe.py --out ../data/can_tickers_swing_universe --input ../data/can_tickers_full

# Ticker metadata builder (run from repo root)
python py/tickers_info.py --input data/can_tickers --out out/can_tickers_info.csv

# Wheel strategy screener (saves wheel_candidates.csv etc. in py/)
cd py && python wheel_screener.py

# Download full Canadian market ticker list from Yahoo (saves to ../data/can_tickers_full)
cd py && python fetch_tickers.py
```

## Architecture

Four independent scripts, no shared module. Each is self-contained with its own config dataclass:

### `py/swing_universe.py` — Swing Universe Builder
Finds Canadian stocks suitable for 1–3 week swing trades. Pipeline:
1. Loads tickers from a text file (`data/can_tickers` or `data/can_tickers_full`)
2. Downloads 1y daily OHLCV history in batches via `yf.download()` (batch_size=80, with sleep between batches to avoid rate limits)
3. Per-ticker: computes ATR-14, dollar volume, SMA50/SMA200, worst 1-day return, SMA50 slope (normalized by price), relative strength vs benchmark (XIU.TO), volume trend
4. Applies hard filters (`Thresholds`): price, dollar volume, ATR%, worst-day drop, above SMA50, stale data
5. Scores passing tickers: liquidity (log scale), trend alignment, RS vs benchmark, volume trend, ATR/worst-day penalties
6. Outputs: `data/can_tickers_swing_universe` (one per line), `out/can_tickers_swing_one_line` (comma-separated), `out/can_tickers_rejected.csv`

Entry points: `run_universe_builder(cfg: UniverseBuilderConfig)` or `__main__` with `--out`, `--out-one-line`, `--out-rejected`, `--input` args.

### `py/wheel_screener.py` — Wheel Options Screener
Screens CA + US stocks for the Wheel Strategy (sell Cash-Secured Puts → Covered Calls). Pipeline:
1. Iterates a hardcoded universe (`CANADIAN_STOCKS` + `US_STOCKS` dicts, 130+ tickers)
2. Per-ticker: fetches fundamentals via `yf.Ticker.info`, options chain (finds best CSP in 20–50 DTE window), earnings proximity
3. Scores across 6 pillars: profitability (30pts), FCF quality (10pts), balance sheet (20pts), valuation (25pts), growth (10pts), options chain quality (20pts)
4. Assigns tiers: 🟢 STRONG / 🟡 SOLID / 🟠 WATCH / 🔴 SKIP
5. Saves `wheel_candidates.csv`, `wheel_all_screened.csv`, `wheel_candidates.json` (relative to cwd)

Entry point: `run_screener()` — configurable via keyword args at bottom of file.

### `py/tickers_info.py` — Ticker Metadata Builder
Fetches `yf.Ticker.info` for each ticker in a list and writes a CSV with exchange, company name, sector, volume, market cap, price, and spread estimate. Spread uses `ask - bid` when available, falls back to `fallback_spread_pct * price` (default 1%).

Entry points: `run_ticker_info_builder(cfg: TickerInfoConfig)` or CLI with `--input`/`--out` args.

### `py/fetch_tickers.py` — Canadian Market Download
Uses `yf.screen(EquityQuery("eq", ["region", "ca"]))` to page through all Canadian tickers and writes them to `data/can_tickers_full`. No config dataclass — just a script.

## Data files

- `data/can_tickers` — curated hand-maintained list of Canadian tickers (Yahoo format: `.TO`, `.V`, `.CN`, `.NE`)
- `data/can_tickers_full` — full Yahoo Canadian market dump generated by `fetch_tickers.py`
- `data/can_tickers_value_universe` — pre-filtered universe for value screening, generated by `value_universe.py`
- `data/can_tickers_swing_universe` — pre-filtered universe for swing trading, generated by `swing_universe.py`
- `out/` — generated output; committed selectively

## Key design notes

- `auto_adjust=True` is required for all `yf.download()` calls to prevent split/dividend artifacts in SMA, ATR, and return calculations.
- `yf.download()` with multiple tickers returns a `pd.MultiIndex` column DataFrame (level 0 = ticker); single-ticker batches return a flat DataFrame — both paths are handled explicitly in `swing_universe.py`.
- `yfinance` returns `debtToEquity` as a percentage (e.g., 150 = 1.5×); `wheel_screener.py` divides by 100 after fetching.
- Benchmark data (XIU.TO) is fetched once before the main batch loop in `swing_universe.py` and aligned per-ticker using `close.align(bench_close, join="inner")`.
